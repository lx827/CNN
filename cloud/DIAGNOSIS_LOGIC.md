# 诊断参数配置与默认诊断逻辑说明

> 本文档说明设备机械参数（轴承/齿轮）如何影响诊断行为，以及"默认诊断逻辑"的触发条件。

---

## 一、参数配置入口

前端路径：`设备管理 → 设备配置`

可配置字段：
- **轴承参数** `bearing_params`：`{n, d, D, alpha}`
  - `n`：滚动体个数
  - `d`：滚动体直径 (mm)
  - `D`：轴承节径 (mm)
  - `alpha`：接触角 (°)
- **齿轮参数** `gear_teeth`：`{input, output}`
  - `input`：主动轮齿数
  - `output`：从动轮齿数

支持两种存储格式（后端自动兼容）：
- **设备级**（旧格式）：`{input: 18, output: 27}`
- **通道级**（新格式）：`{"1": {input: 18}, "2": {input: 18}}`

---

## 二、诊断逻辑规则

### 2.1 核心原则

| 场景 | 轴承诊断 | 齿轮诊断 |
|------|---------|---------|
| **配置了轴承参数，未配置齿轮参数** | ✅ 用配置参数执行 | ❌ 跳过 |
| **未配置轴承参数，配置了齿轮参数** | ❌ 跳过 | ✅ 用配置参数执行 |
| **两者都配置了** | ✅ 用配置参数执行 | ✅ 用配置参数执行 |
| **两者都没配置** | ✅ **默认诊断逻辑** | ✅ **默认诊断逻辑** |

> **默认诊断逻辑**：当设备完全没有配置任何机械参数时，系统不会跳过诊断，而是自动采用一组内置的默认参数执行全套诊断，确保 Dashboard 和实时分析页面始终有数据展示。

### 2.2 默认参数值

```python
# 默认轴承参数：6205-2RS 深沟球轴承
DEFAULT_BEARING_PARAMS = {"n": 9, "d": 7.94, "D": 39.04, "alpha": 0}

# 默认齿轮参数：常见齿轮箱速比
DEFAULT_GEAR_TEETH = {"input": 18, "output": 27}
```

### 2.3 为什么需要默认诊断逻辑

- **后台分析 worker** 必须保证 Dashboard 的故障分布、健康度评分始终可用
- **实时分析接口** 用户在 DataView 点击"运行诊断"时期望看到结果，而不是"No Data"
- **新设备** 创建时默认不预填机械参数（避免用户误以为已配置），但诊断仍然可以运行

## 三、各接口行为对照

| 接口 | 路径 | 无配置时的行为 |
|------|------|---------------|
| 后台综合分析 | `analyzer.py analyze_device` | 默认参数做全套 |
| 综合分析 | `/api/data/{...}/analyze` | 默认参数做全套 |
| 全算法对比 | `/api/data/{...}/full-analysis` | 默认参数做全套 |
| 齿轮诊断 | `/api/data/{...}/gear` | 默认齿轮参数 |
| 包络分析 | `/api/data/{...}/envelope` | 默认轴承参数 |

---

## 四、注意事项

1. **默认参数仅供参考**：真实风机请务必在设备配置页面录入准确的轴承型号和齿轮齿数，否则诊断结果可能偏离实际。

2. **参数有效性判断**：后端通过 `_has_valid_bearing()` / `_has_valid_gear()` 判断参数是否有效。只要 `n/d/D`（轴承）或 `input`（齿轮）任一关键字段缺失或无效，即视为"未配置"。

3. **离线设备不诊断**：无论是否配置了参数，设备离线后所有实时分析接口返回 `400 设备当前离线`，后台分析 worker 自动跳过离线设备。

4. **前端空数据提示**：DataView 的全算法分析页面在表格无数据时会提示"未配置XX参数"，但如果是因为设备刚创建且未配置，后端会自动使用默认参数，正常情况下不会触发该提示。
