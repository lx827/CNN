## 1. 项目概述

### 1.1 背景与意义

风机齿轮箱是风力发电机组的核心传动部件，其运行状态直接影响机组的发电效率和安全性。据统计，齿轮箱故障占风机总故障的 20%~30%，且维修成本占总运维成本的 40% 以上。传统运维模式多为"事后维修"或"定期检修"，存在以下问题：

1. **事后维修**：故障已发生才维修，导致长时间停机，经济损失巨大。
2. **定期检修**：不管设备状态如何都检修，造成"过度维修"浪费人力物力。
3. **依赖人工经验**：传统振动分析需要专业人员手动解读频谱，效率低且主观性强。

本项目开发了一套**风机齿轮箱智能故障诊断系统**，通过多算法集成诊断引擎，实现对轴承和齿轮故障的自动识别、健康度评估和维护建议生成。系统已部署在阿里云 2 核 2G 服务器上，支持 Web 端实时监测和故障预警。

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| **多算法集成** | 实现 13 种轴承诊断方法 + 2 种齿轮诊断方法的统一调度 |
| **D-S 证据融合** | 通过 Dempster-Shafer 证据理论融合多方法结果，降低误报率 |
| **连续健康度评分** | 用 0~100 连续评分替代固定阈值跳变，实现趋势监控 |
| **工程落地** | 在 2GB 内存服务器上实时运行，支持 Web 端访问 |
| **三种数据集验证** | 在 HUSTbear（恒速轴承）、CW（变速轴承）、WTgearbox（行星齿轮）三个公开数据集上验证 |

### 1.3 系统核心能力

- **故障检测**：对轴承外圈/内圈/滚动体故障、齿轮断齿/缺齿/裂纹/磨损进行检测
- **故障分类**：区分 5 类轴承故障（健康/球故障/内圈/外圈/复合）和 5 类齿轮故障（健康/断齿/缺齿/裂纹/磨损）
- **健康度评估**：输出 0~100 连续健康度评分，支持趋势监控
- **维护建议**：根据诊断结果生成具体的维护建议文本

### 3.2 核心算法原理

#### 3.2.1 包络分析（Envelope Analysis）

**物理原理**：轴承局部缺陷产生周期性冲击脉冲，该脉冲激发结构高频共振。共振信号被故障脉冲序列调幅（AM），包络分析通过解调提取低频调制信号（故障脉冲序列），抑制高频载波。

**完整算法流程**：

1. **带通滤波**：选择共振频带（通常 2~20 kHz）
   $$y(t) = \text{BandPass}[x(t); f_c, \Delta f]$$

2. **希尔伯特变换**：构造解析信号
   $$z(t) = y(t) + j\mathcal{H}[y(t)]$$

3. **包络提取**：
   $$e(t) = |z(t)| = \sqrt{y^2(t) + \mathcal{H}^2[y(t)]}$$

4. **低通滤波**：
   $$e_{LP}(t) = \text{LowPass}[e(t); f_{cut}]$$

5. **FFT 包络谱**：
   $$E(f) = |\mathcal{F}[e_{LP}(t)]|$$

**故障判据**：

- 外圈故障：包络谱中出现 $n \times BPFO$ 谐波族（$n=1,2,3,\dots$）
- 内圈故障：BPFI 两侧出现以轴频 $f_r$ 为间隔的边带 $BPFI \pm k f_r$
- 滚动体故障：$2 \times BSF$ 两侧可能出现保持架频率 FTF 的边带

#### 3.2.2 Fast Kurtogram

**物理意义**：谱峭度度量信号各频率分量偏离高斯分布的程度，对瞬态冲击极度敏感。Fast Kurtogram 通过树状滤波器组自动寻找信号中最峭度的频带，相当于让算法自己找到故障冲击最明显的频段。

**数学定义**：
$$K(f, \Delta f) = \frac{\langle|X|^4\rangle}{\langle|X|^2\rangle^2} - 2$$

**算法流程**：

1. 构建准解析滤波器组（2/3/4/6 分段）
2. 树状分解，第 $l$ 层产生 $2^l$ 个子带
3. 计算各子带峭度 $K_{l,i}$
4. 构建 Kurtogram 图，选取全局最大峭度对应的 $(f_c^*, \Delta f^*)$
5. 对最优子带执行包络分析

**工程改进**：

- **Protrugram**：用包络谱幅值的峭度替代时域峭度，抗非高斯噪声更强
- **MED+SK**：先用最小熵解卷积锐化冲击，再计算谱峭度

#### 3.2.3 倒频谱预白化（CPW）

**核心问题**：齿轮啮合频率、轴频等确定性离散频率会掩盖轴承故障的弱冲击成分。

**完整数学流程**：

1. FFT：$X(f) = \mathcal{F}[x(t)]$
2. 功率谱对数：$\ln|X(f)|$
3. 倒频谱变换：$C(\tau) = \mathcal{F}^{-1}\{\ln|X(f)|\}$
4. 倒频谱编辑（Comb Lifter）：去除齿轮/轴频对应的倒频谱线
5. 重构幅值谱：$\ln|X_{edit}(f)| = \mathcal{F}[C_{edit}(\tau)]$
6. 结合原始相位重构信号
7. 对 $x_{CPW}(t)$ 执行包络分析

#### 3.2.4 D-S 证据融合

Dempster-Shafer 证据理论将多种诊断算法的结果融合为综合故障概率分布。

**识别框架**：
$$\Theta = \{\text{轴承外圈故障, 轴承内圈故障, 轴承滚动体故障, 齿轮磨损, 齿轮裂纹, 齿轮断齿, 正常}\}$$

**基本概率分配（BPA）**：
每个诊断方法的结果映射为 BPA：

- confidence > 0.55 且 abnormal → 强证据，分配质量到具体故障
- confidence 0.2~0.55 → 弱证据，部分质量给故障，更多给 Θ
- confidence < 0.2 → Θ 占主导

**Dempster 组合规则**：
$$m_{12}(A) = \frac{\sum_{B \cap C = A} m_1(B)m_2(C)}{1 - K}$$
$$K = \sum_{B \cap C = \emptyset} m_1(B)m_2(C)$$

**高冲突修正（Murphy 平均法）**：
当冲突系数 $K > 0.8$ 时，标准 Dempster 规则不可靠，切换到 Murphy 平均法：

1. 计算所有 BPA 的平均 BPA
2. 用平均 BPA 与自身组合 $n-1$ 次

#### 3.2.5 齿轮诊断指标

**SER（边频带能量比）**：
$$SER = \frac{\sum_{i=1}^{6}(A_{SB_i}^+ + A_{SB_i}^-)}{A(f_{mesh})}$$
健康齿轮 $SER < 1$；故障齿轮边频增加，$SER > 1$。

**FM4（局部故障指标）**：
$$FM4 = \frac{N\sum(d_i - \bar{d})^4}{[\sum(d_i - \bar{d})^2]^2}$$
健康 $\approx 3$；局部故障 $> 5\sim10$。

**FM0（粗故障指标）**：
$$FM0 = \frac{PP}{\sum_{i=1}^{n} A(f_i)}$$
健康 $< 3\sim5$；严重故障 $> 10\sim20$。

**CAR（倒频谱幅值比）**：
$$CAR = \frac{\text{mean}[\sum C_{peak}^i]}{\text{mean}_{\tau} C_x(\tau)}$$
$CAR > 1$ 且持续上升指示齿轮劣化。

### 3.3 健康度评分

健康度评分采用**连续衰减扣分**机制，替代传统固定阈值跳变：

$$H = 100 - \sum w_i \cdot \text{sigmoid}(f_i, \theta_i)$$

**扣分路径**：

- 时域冲击证据（kurtosis > 12 或 crest > 10）：扣 20~30 分
- BPFO/BPFI/BSF 显著（SNR > 4.5）：扣 15~25 分
- 动态基线偏离（kurt_mad_z > 3）：扣 10~20 分
- 齿轮指标异常（SER > 3 或 FM4 > 10）：扣 10~15 分

**状态判定**：

- $H \geq 85$ → normal
- $60 \leq H < 85$ → warning
- $H < 60$ → fault

## 6. 创新点与工程价值

### 6.1 方法创新

#### 6.1.1 多算法集成 + D-S 证据融合

传统系统通常只用一种诊断方法（如包络分析），本系统集成 13 种轴承方法 + 2 种齿轮方法，通过 D-S 证据理论融合结果。

**优势**：

- 单一方法可能误判，融合可降低误报率
- 不同工况下最优方法不同，集成可自适应选择
- D-S 融合输出置信度，便于不确定性量化

#### 6.1.2 连续健康度评分

传统系统用固定阈值跳变（正常/故障），本系统输出 0~100 连续评分。

**优势**：

- 像体温计一样平滑反映设备状态
- 支持趋势监控和预测性维护
- 避免健康度忽上忽下

#### 6.1.3 参数驱动的分析跳过逻辑

- 仅配置轴承参数 → 只做轴承诊断，跳过齿轮（避免齿轮统计指标误报）
- 仅配置齿轮参数 → 只做齿轮诊断，跳过轴承
- 都未配置 → 跑统计指标（SER/CAR/阶次峭度）
- 都配置 → 综合（轴承+齿轮）全跑

### 6.2 工程创新

#### 6.2.1 2GB 服务器适配

| 优化项 | 说明 |
|--------|------|
| 线程池限制 | `ThreadPoolExecutor(max_workers=2)`，绝对不增加 |
| 后台分析串行化 | `asyncio.Semaphore(1)`，一次只分析一个批次 |
| 信号截断 | 最多处理 5 秒数据，防止 OOM |
| 按需计算 | DataView 的 FFT/STFT/包络等接口点击后才计算 |

#### 6.2.2 诊断缓存策略

- 缓存键为 `(device_id, batch_index, channel, denoise_method)`
- 不同去噪方法的结果独立保存，互不覆盖
- 支持按通道和去噪方法分版本查询

#### 6.2.3 云端动态配置

- 边端启动时从云端拉取配置（采样率、时长、压缩开关等）
- 运行中每 30 秒刷新一次
- 前端 Settings 页面修改后，最长 30 秒内同步到边端
- 支持批量应用到所有设备

### 6.3 与深度学习的对比

| 维度 | 本系统 | 深度学习 |
|------|--------|---------|
| 数据需求 | 无需大量标注数据 | 需要大量标注数据 |
| 可解释性 | 强（D-S 融合有置信度，诊断逻辑透明） | 弱（黑盒） |
| 计算资源 | 2GB 服务器可运行 | 需要 GPU 或高性能服务器 |
| 部署成本 | 低（阿里云 2 核 2G） | 高（GPU 服务器） |
| 诊断逻辑 | 透明（每种方法原理明确） | 不透明（权重难以解释） |

> **定位差异**：我们做的是"工业落地型"系统，而非"实验室精度型"。在超大数据量场景下，深度学习可能上限更高，但本系统在工程实用性、可解释性、部署成本方面具有明显优势。

## 7. 不足与改进计划

### 7.1 当前不足

#### 7.1.1 齿轮诊断 Ensemble 表现差

**问题**：在 WTgearbox 数据集上，Ensemble 二分类准确率仅 38.75%，五类分类仅 18.75%。

**原因**：

1. 行星齿轮箱的频域指标（SER/CAR/sideband）在健康和故障之间重叠严重
2. D-S 融合的 BPA 分配中，齿轮 fault_indicators 的 warning/critical 标记未能正确映射到故障概率
3. 齿轮方法（Standard/Advanced）返回的 fault_indicators 结构与轴承方法不同，D-S 融合时未能有效利用

**改进计划**：

1. 修复齿轮 BPA 分配逻辑，将 warning/critical 正确映射到故障概率
2. 增加行星齿轮箱专用指标（如窄带包络阶次分析、VMD 幅频联合解调）到 D-S 融合
3. 调整行星箱阈值，使用多指标联合判断而非单一阈值

#### 7.1.2 轴承五类分类准确率低

**问题**：在 HUSTbear 数据集上，Ensemble 五类分类准确率仅 23.23%。

**原因**：

1. 诊断引擎输出的是 `health_score` + `fault_indicators`（故障频率 SNR），不是直接的 5 类分类器
2. 标签推断逻辑（从 fault_indicators 映射到 5 类标签）存在信息损失
3. 特别是球故障（ball）和内圈（inner）容易混淆，因为两者的故障频率系数接近

**改进计划**：

1. 增加直接的多分类器（如在提取的特征向量上训练决策树/SVM）
2. 改进标签推断逻辑，使用 SNR 比值而非单一最高 SNR
3. 增加球故障专用检测方法（如 2×BSF 谐波族验证）

#### 7.1.3 变速工况阶次跟踪精度受限

**问题**：在 CW 数据集上，三分类准确率仅 33.33%。

**原因**：

1. CW 数据集全部为变速工况，故障频率随时间变化
2. 阶次跟踪算法在短信号（5 秒）上精度受限
3. 传统 BPFO/BPFI 匹配方法难以区分内圈/外圈

**改进计划**：

1. 增加变速阶次跟踪算法（STFT + 等相位重采样）
2. 增加循环频率估计，自动适应转速变化
3. 增加阶次谱 + D-S 融合的联合诊断
