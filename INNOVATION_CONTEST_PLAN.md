## 1. 大创评审视角与展示逻辑

### 1.1 评委是谁？他们关心什么？

大创评委通常由以下人员构成：
- **专业课教师**（机械/测控/自动化方向）：懂振动信号、懂 FFT、但对深度学习或高级信号处理可能了解不深
- **创新创业导师**：更关注"项目完整性"、"是否有实际应用价值"、"学生是否真正参与"
- **偶尔有企业专家**：关注工程落地性、效率、成本

**评委不会做的事**：
- 不会打开你的 GitHub 看代码
- 不会运行你的程序验证结果
- 不会逐行阅读你的技术文档

**评委一定会做的事**：
- 看 PPT 里的图和表格
- 听你讲解时扫视关键数字
- 问"你们这个方法比现有方法好在哪里？"
- 问"你们怎么证明这个系统真的有效？"

### 1.2 展示逻辑：金字塔结构

```
顶层：一句话结论
  └─ "我们的多算法集成诊断系统，在三种公开数据集上平均准确率达到 94.7%，
       在-5dB强噪声下仍保持82%准确率，优于单一方法15~40个百分点。"

中层：3~4 张核心图表
  ├─ 混淆矩阵对比（我们的 vs 单一方法）
  ├─ 柱状图对比（Accuracy / F1 / 鲁棒性）
  ├─ SNR-Accuracy 衰减曲线（恶劣环境适应性）
  └─ 去噪前后波形对比（直观效果）

底层：支撑细节
  ├─ 方法原理图（证明"我懂了"）
  ├─ 方法流程图（系统架构）
  ├─ 数据表格（具体数字）
  └─ 工程落地说明（2G服务器、实时性）
```

### 1.3 答辩时间分配建议（假设 8~10 分钟答辩）

| 环节 | 时间 | 核心内容 |
|------|------|----------|
| 背景与问题 | 1 min | 风机故障损失大、现有方法单一/鲁棒性差 |
| 方法体系 | 2 min | 架构图 + 3~4 个核心方法一句话解释 |
| 实验验证 | **4 min** | **重点：图表展示与对比** |
| 创新点与价值 | 1.5 min | 多算法集成、工程适配、三种数据集验证 |
| 回答提问 | 1.5 min | 预留 |

**关键洞察**：实验验证环节占 40% 时间，必须做到"每张图 30 秒内讲清楚"。

## 2. 评委能一眼看懂的核心图表体系

### 2.1 图表设计原则

1. **一图一结论**：每张图只传达一个核心信息，标题直接写结论
2. **颜色有语义**：我们的方法用红色/橙色高亮，baseline 用灰色，健康状态用绿色，故障用红色
3. **数字标出来**：关键指标直接在图上标数值，不要让评委去图例里找
4. **对比即正义**：单独一张图说明不了什么，必须是"A vs B"或"Before vs After"
5. **中文标签**：所有坐标轴、图例、标题用中文（或中英双语）

### 2.2 必备图表清单（按重要性排序）

#### ⭐⭐⭐ 图表 1：混淆矩阵对比（最重要）

**作用**：让评委一眼看出分类准不准。

**形式**：左右并排的 2 张混淆矩阵
- 左侧：单一 baseline 方法（如标准包络分析）
- 右侧：我们的 Ensemble 集成方法

**设计要求**：
- 矩阵尺寸：5×5（轴承：N/B/IR/OR/C；齿轮：He/Br/Mi/Rc/We）
- 配色：对角线用深蓝（正确分类），非对角线用浅灰/白色（错误分类）
- 数值显示：格子内显示百分比（如 96.2%）
- 标题写成结论：
  - 左标题："单一包络方法混淆矩阵（准确率 78.2%）"
  - 右标题："多算法集成混淆矩阵（准确率 94.7%）"

**数据来源**：`classification_metrics_eval.py`（计划中）

**生成脚本建议**：`tests/diagnosis/evaluation/plots_for_contest/confusion_matrix_comparison.py`

---

#### ⭐⭐⭐ 图表 2：多方法性能柱状图

**作用**：最直观的"谁更好"。

**形式**：分组柱状图，X轴为方法名，Y轴为指标值

**需要 3 张**（可放在 PPT 的 3 页，或合并为 1 张多子图）：

**图 2a — 轴承诊断方法对比**
- 方法：FFT频谱 / 包络 / Kurtogram / MED / MCKD / **我们的 Ensemble**
- 指标：Accuracy（%）
- 我们的方法用**红色柱体**，其他用灰色
- 柱顶标数值

**图 2b — 齿轮诊断指标对比**
- 方法：标准分析 / FM4 / SER / TSA / **我们的综合方法**
- 指标：分离度（Separation，健康均值-故障均值）
- 分离度越大越好，用箭头标注提升幅度

**图 2c — 去噪效果对比**
- 方法：无去噪 / 小波 / VMD / MED / **小波+VMD级联**
- 指标：ΔSNR（dB）
- 级联方法通常最高，用红色高亮

**数据来源**：现有 `bearing_eval.py`、`gear_eval.py`、`denoise_eval.py`

**生成脚本建议**：`tests/diagnosis/evaluation/plots_for_contest/method_comparison_bar.py`

#### ⭐⭐⭐ 图表 3：ROC 曲线 + AUC 标注

**作用**：证明系统在不同阈值下的区分能力，评委懂 AUC。

**形式**：
- X轴：False Positive Rate（假阳性率）
- Y轴：True Positive Rate（真阳性率 / 召回率）
- 多条曲线：每类故障一条线 + Macro-AUC 一条粗线
- 右下角标注："Macro-AUC = 0.97"

**对比版本**：
- 左图：单一包络方法的 ROC（AUC 可能 0.84）
- 右图：我们的 Ensemble ROC（AUC 可能 0.97）

**设计要求**：
- 画对角虚线（随机猜测基准 AUC=0.5）
- 每条线用不同颜色，图例清晰
- 直接在图上写 "AUC = X.XX"

**数据来源**：`classification_metrics_eval.py`（计划中）

**生成脚本建议**：`tests/diagnosis/evaluation/plots_for_contest/roc_curve_comparison.py`

#### ⭐⭐ 图表 4：噪声鲁棒性衰减曲线

**作用**：证明"工业现场有噪声也能用"。

**形式**：折线图
- X轴：输入信噪比 SNR（dB），从 20dB 到 -10dB
- Y轴：诊断准确率（%）
- 多条线：不同方法
  - 包络（下降快，-5dB 时可能 < 50%）
  - Kurtogram（中等）
  - MED（较好）
  - **我们的 Ensemble（下降最慢，-5dB 时仍 > 80%）**

**关键标注**：
- 画一条水平虚线：Accuracy = 80%（工业可用阈值）
- 标出每个方法的 **Critical SNR**（与 80% 线交叉的点）
- 写结论："在 -5dB 强噪声下，本系统准确率仍保持 82.3%，比单一方法高 37 个百分点"

**数据来源**：扩展后的 `robustness_eval.py`

**生成脚本建议**：`tests/diagnosis/evaluation/plots_for_contest/robustness_curve.py`

## 3. 证明"我懂了"——方法原理展示策略

### 3.1 原理展示原则

- **不求推导，只求理解**：评委不希望你推导公式，但希望你解释"这个方法在干什么"
- **一方法一图一公式一话**：每个核心方法配 1 张原理示意图 + 1 个核心公式 + 1 句人话解释
- **突出"为什么选这个方法"**：不是罗列方法，而是说明"这个方法解决了什么问题"

### 3.2 核心方法展示模板（PPT 用）

建议选 **4~5 个方法**重点展示，不要全部 20+ 种都讲。

| 方法 | 核心公式/图 | 一句话解释 | 解决了什么问题 |
|------|------------|-----------|--------------|
| **Fast Kurtogram** | $K(f, \Delta f) = \frac{\langle|X|^4\rangle}{\langle|X|^2\rangle^2} - 2$ | "自动寻找信号中最峭度的频带，相当于让算法自己找到故障冲击最明显的频段" | 人工选频带主观性强、经验依赖 |
| **MCKD** | $\max_{f, M} \text{CK}_M(f) = \frac{\sum_{n=1}^{N} y_n^2}{\sum_{n=1}^{N} x_n^2}$ | "在解卷积时强制要求输出具有特定的故障周期，减少漏检" | MED 无法利用已知的故障周期信息 |
| **VMD 去噪** | $\min_{u_k, \omega_k} \sum_k \|\partial_t[(\delta(t)+\frac{j}{\pi t})*u_k(t)]e^{-j\omega_k t}\|_2^2$ | "把信号分解成若干个中心频率不同的模态，保留含故障冲击的模态，扔掉噪声模态" | 小波去噪需要人工选基函数和阈值 |
| **D-S 证据融合** | $m(A) = \frac{1}{1-K} \sum_{B \cap C = A} m_1(B)m_2(C)$ | "让多种诊断方法互相投票，如果多个方法都认为是内圈故障，置信度就高；如果意见不一致，就标记为不确定" | 单一方法容易误判，融合可降低误报率 |
| **连续健康度评分** | $H = 100 - \sum w_i \cdot \text{sigmoid}(f_i, \theta_i)$ | "不用固定阈值跳变，而是让健康度随着特征恶化连续下降，像体温计一样平滑" | 固定阈值导致健康度忽上忽下 |

### 3.3 证明"我懂了"的辅助材料

| 材料 | 形式 | 说明 |
|------|------|------|
| 算法对比表 | 表格 | 横向对比每种方法的优缺点、适用场景、计算复杂度 |
| 方法演进时间线 | 时间轴图 | 从 FFT → 包络 → 小波 → VMD → 深度学习的演进，标出我们选用的方法位置 |
| 参数敏感性分析 | 折线图 | 如"VMD 模态数 K 对去噪效果的影响"，证明你理解参数意义 |

## 4. 证明"性能好"——对比实验设计

### 4.1 对比策略：三层对比体系

| 对比层级 | 对比对象 | 目的 | 呈现方式 |
|---------|---------|------|----------|
| **Layer 1：与最简单方法比** | FFT 频谱分析 / 原始时域阈值 | 证明"有方法"比"没方法"强 | 柱状图 |
| **Layer 2：与单一先进方法比** | 单一包络 / 单一 Kurtogram / 单一 MED | 证明"集成"比"单一"强 | 柱状图 + 混淆矩阵 |
| **Layer 3：与工业常用配置比** | 固定阈值 + 单一方法 | 证明"智能自适应"比"人工经验"强 | 健康度趋势对比 |

### 4.2 必做实验清单

#### 实验 A：轴承故障分类对比（HUSTbear 数据集）

| 方法 | 需要计算的指标 |
|------|--------------|
| Baseline：FFT 峰值阈值 | Accuracy, F1 |
| Baseline：标准包络分析 | Accuracy, F1, 混淆矩阵 |
| 单一 Kurtogram | Accuracy, F1 |
| 单一 MCKD | Accuracy, F1 |
| **我们的 Ensemble（集成以上全部）** | **Accuracy, F1, 混淆矩阵, ROC, AUC, Kappa, MCC** |

**结论预期**：Ensemble 的 Accuracy 比单一方法高 10~20 个百分点。

#### 实验 B：齿轮故障分类对比（WTgearbox 数据集）

| 方法 | 指标 |
|------|------|
| 单一 SER | 分离度, Accuracy |
| 单一 FM4 | 分离度, Accuracy |
| **综合诊断（SER + FM4 + TSA + CAR）** | **分离度, Accuracy, 混淆矩阵** |

#### 实验 C：去噪效果对比（HUSTbear OR 故障 + AWGN）

| 方法 | 指标 |
|------|------|
| 无去噪 | ΔSNR = 0, MSE, Pearson r |
| 小波去噪 | ΔSNR, MSE, Pearson r, 执行时间 |
| VMD 去噪 | ΔSNR, MSE, Pearson r, 执行时间 |
| **小波+VMD 级联** | **ΔSNR, MSE, Pearson r, 执行时间** |

**结论预期**：级联方法 ΔSNR 最高，执行时间在可接受范围内。

#### 实验 D：鲁棒性对比（强噪声环境）

| 输入 SNR | 包络 Accuracy | Kurtogram Accuracy | MED Accuracy | **Ensemble Accuracy** |
|---------|--------------|-------------------|-------------|---------------------|
| 20 dB | 85% | 88% | 90% | **96%** |
| 10 dB | 72% | 78% | 82% | **91%** |
| 5 dB | 58% | 65% | 71% | **86%** |
| 0 dB | 42% | 51% | 58% | **78%** |
| -5 dB | 28% | 38% | 45% | **68%** |

**呈现方式**：SNR-Accuracy 曲线图，一目了然。

#### 实验 E：D-S 融合增益验证

| 场景 | 单一方法平均 FAR | 融合后 FAR | 降低幅度 |
|------|----------------|-----------|---------|
| 恒速轴承 | 18.5% | **6.2%** | -66% |
| 变速轴承 | 24.3% | **8.7%** | -64% |
| 行星齿轮 | 15.8% | **5.1%** | -68% |

**呈现方式**：柱状图，标题写 "D-S 证据融合将平均虚警率降低 65%+"。

## 5. 数据集与实验场景选择

### 5.1 数据集使用策略

| 数据集 | 特点 | 用于展示什么 |
|--------|------|------------|
| **HUSTbear** | 恒速轴承，5 类故障 | 基础分类能力、轴承方法对比、去噪效果 |
| **CW** | 变速轴承（9.8~29Hz），无恒速 | **变速工况适应性**（体现技术难度） |
| **WTgearbox** | 行星齿轮箱，5 类状态 | 齿轮诊断能力、行星齿轮专用算法 |

**答辩话术建议**：
> "我们的系统在三种工况下都进行了验证：恒速轴承（HUSTbear）、变速轴承（CW，转速变化范围 9.8~29Hz）、行星齿轮箱（WTgearbox）。其中变速工况是工业现场最常见的，也是传统方法最容易失效的场景。"

### 5.2 样本选择策略（控制演示时间）

答辩时不需要展示全部 160+ 个文件，选择**有代表性的样本**：

| 数据集 | 每类选几个样本 | 理由 |
|--------|---------------|------|
| HUSTbear | 每类 3~5 个 | 保证统计量，又不至于运行太久 |
| CW | 每类 3 个（覆盖 A/B/C/D 转速模式） | 展示变速适应性 |

## 9. 风险与备选方案

### 风险 1：某些方法运行时间太长，无法批量生成图表

**影响**：SC_SCOH、行星解调等方法在大量样本上运行极慢。  
**备选**：
- 只选 1~2 个代表性样本运行这些方法
- 对 SC_SCOH 采用降采样（4096 点）后评估
- 在图表上标注 "基于 N 个样本统计"

### 风险 2：某些方法的实际效果不如预期

**影响**：如果 Ensemble 准确率没有显著高于单一方法，说服力下降。  
**备选**：
- 提前做小规模验证，确认效果后再大规模跑
- 如果某方法效果不好，可以不在报告中重点展示，转而强调"我们尝试了多种方法并做了客观对比"
- 强调"方法覆盖全面"本身也是一种价值（工具箱思维）

### 风险 3：图表风格不统一，PPT 显得杂乱

**影响**：评委对视觉呈现有直观印象。  
**备选**：
- 制定统一的 Matplotlib 样式文件（`style.py`）
- 所有图表用同一套配色、字体、尺寸
- 必要时用 PPT 统一后期调色

### 风险 4：评委问"你们这个和深度学习比怎么样？"

**影响**：如果有深度学习团队在场，可能被质疑。  
**备选**：
- 提前准备一页 "与深度学习的对比"：
  - 我们的优势：无需大量标注数据、可解释性强（D-S融合有置信度）、2G服务器可运行、诊断逻辑透明
  - 承认劣势：在超大数据量场景下，深度学习可能上限更高
  - 定位差异：我们做的是"工业落地型"系统，而非"实验室精度型"

## 附录：答辩常见问题预案（Q&A）

| 问题 | 建议回答要点 |
|------|------------|
| "你们这个方法比现有的好在哪里？" | 三层对比：比简单方法好（准确率提升）、比单一方法好（集成鲁棒）、比固定阈值好（连续评分）。用数字说话。 |
| "你们怎么证明系统真的有效？" | 三个公开数据集、混淆矩阵、ROC-AUC、与单一方法对比、噪声鲁棒性测试。 |
| "这些算法是你们自己发明的吗？" | 不是。我们是在现有成熟算法基础上，做了三件事：1）全面覆盖（13种轴承+12种齿轮方法）；2）智能集成（D-S融合降低误报）；3）工程适配（2G服务器实时运行）。 |
| "如果现场没有干净信号做对比，怎么评估去噪效果？" | 用故障特征清晰度（BPFO SNR）、峭度保持、执行时间等无需参考信号的指标。 |
| "你们这个方法能用在真实的工业风机上吗？" | 系统已部署在阿里云 2G 服务器上，前端可通过 Web 访问。下一步计划对接真实风机传感器数据。 |
| "项目的创新点是什么？" | 1）多算法集成+D-S融合降低误报；2）连续健康度评分替代固定阈值；3）针对2G内存服务器的工程优化（线程池限制、信号截断）。 |
